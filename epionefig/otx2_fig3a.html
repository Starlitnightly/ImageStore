OTX2 case study — Fig 3a¶

UCSC-browser snapshot of OTX2 CUT&RUN at the ZNF-cluster region (chr3:137.7-143.3 Mb) in 4C embryos (rep1, rep2), OTX2-KD 4C, 8C, 4C ATAC and primed hESC OTX2 ChIP (Tsankov et al.).

Paper: Wang, Q. et al. Maternal factor OTX2 regulates human embryonic genome activation and early development. Nature Genetics 57, 2772–2784 (2025). 10.1038/s41588-025-02350-8

Data availability¶

Primary data from the paper (Wang et al. 2025):

  • Raw sequencing: GSA-Human HRA006621 (controlled-access, not used here).
  • Processed bigwigs / peaks / counts: OMIX OMIX006476 (link). Contains OTX2 CUT&RUN bigwigs (4C / 8C / KD / hESC), ATAC-seq bigwigs, and peak BEDs.
  • Supplementary tables S1-S7: downloaded from the Nature article HTML (stage-activated gene lists, DEG tables, 4C OTX2 CUT&RUN peaks).

External data:

  • RNA / Ribo at pre-implantation stages (Zou et al. 2022 Science): GEO GSE197265 — processed bigwigs GSE197265_h_{GV,MII,1C,2C,4C,8C,ICM}_rep12_merge_RNA_10bp_rpkm.bw.
  • 8C H3K4me3 / H3K27ac (Xia et al. 2019 Science, paper ref 82): GEO GSE124718 — GSM3543863/4_hs_8cell_3PN_H3K4me3_rep{1,2}.bedGraph.gz and GSM3820010/11_hs_8cell_3PN_H3K27ac_rep{1,2}.bedGraph.gz (we use 3PN because H3K27ac is only available in 3PN embryos).

%%script false --no-raise-error
# ==========================================================
# CUT&RUN FASTQ -> deduplicated BAM -> RPKM-normalised bigwig
# ==========================================================
# Reference: Wang et al. 2025 Methods ("CUT&RUN assay" section) +
# filename convention in OMIX006476 (`..._pmu_100bp_rpkm_bamCoverage.bw`).
#
# Raw FASTQs are hosted behind controlled access at GSA-Human HRA006621
# (paper Data Availability). For each replicate:
SAMPLE=h_OTX2_day2_CUTRUN_rep1         # one of rep1 / rep2 / KD etc.
FQDIR=/scratch/users/steorra/data/otx2/HRA006621/fastq
BAMDIR=/scratch/users/steorra/data/otx2/OMIX006476/bam
BWDIR=/scratch/users/steorra/data/otx2/OMIX006476
IDX=/scratch/users/steorra/data/hg19/bt2_idx/hg19
mkdir -p $BAMDIR $BWDIR

# 1. Adapter / quality trim
trim_galore --paired --gzip -o $BAMDIR/trim \
    $FQDIR/${SAMPLE}_R1.fastq.gz $FQDIR/${SAMPLE}_R2.fastq.gz

# 2. Bowtie2 alignment (paired, CUT&RUN-optimised)
bowtie2 --end-to-end --very-sensitive --no-mixed --no-discordant \
        -I 10 -X 700 -p 8 -x $IDX \
        -1 $BAMDIR/trim/${SAMPLE}_R1_val_1.fq.gz \
        -2 $BAMDIR/trim/${SAMPLE}_R2_val_2.fq.gz |
    samtools view -bS -q 30 -f 2 - |
    samtools sort -@ 8 -o $BAMDIR/${SAMPLE}.raw.bam

# 3. Picard MarkDuplicates -> "pmu" (picard mark unique) BAM
picard MarkDuplicates \
    I=$BAMDIR/${SAMPLE}.raw.bam \
    O=$BAMDIR/${SAMPLE}.pmu.bam \
    M=$BAMDIR/${SAMPLE}.pmu.metrics REMOVE_DUPLICATES=true
samtools index $BAMDIR/${SAMPLE}.pmu.bam

# 4. Merge biological replicates (rep1 + rep2 -> rep12)
samtools merge -@ 8 -o $BAMDIR/h_OTX2_day2_CUTRUN_rep12.pmu.bam \
    $BAMDIR/h_OTX2_day2_CUTRUN_rep1.pmu.bam \
    $BAMDIR/h_OTX2_day2_CUTRUN_rep2.pmu.bam
samtools index $BAMDIR/h_OTX2_day2_CUTRUN_rep12.pmu.bam

# 5. RPKM bigwig (matches published file name)
bamCoverage -b $BAMDIR/h_OTX2_day2_CUTRUN_rep12.pmu.bam \
    -o $BWDIR/h_OTX2_day2_CUTRUN_rep12_pmu_100bp_rpkm_bamCoverage.bw \
    --binSize 100 --normalizeUsing RPKM -p 8

%%script false --no-raise-error
# ==========================================================
# ATAC-seq FASTQ -> deduplicated, Tn5-shifted BAM -> reads-count bigwig
# ==========================================================
SAMPLE=h_OTX2_ctrl_day2_ATAC_rep1
FQDIR=/scratch/users/steorra/data/otx2/HRA006621/fastq
BAMDIR=/scratch/users/steorra/data/otx2/OMIX006476/bam_atac
BWDIR=/scratch/users/steorra/data/otx2/OMIX006476
IDX=/scratch/users/steorra/data/hg19/bt2_idx/hg19
mkdir -p $BAMDIR $BWDIR

trim_galore --paired --gzip -o $BAMDIR/trim \
    $FQDIR/${SAMPLE}_R1.fastq.gz $FQDIR/${SAMPLE}_R2.fastq.gz
bowtie2 --very-sensitive -X 2000 -p 8 -x $IDX \
        -1 $BAMDIR/trim/${SAMPLE}_R1_val_1.fq.gz \
        -2 $BAMDIR/trim/${SAMPLE}_R2_val_2.fq.gz |
    samtools view -bS -q 30 -f 2 -F 1804 - |
    grep -v chrM |                      # drop mitochondrial reads
    samtools sort -@ 8 -o $BAMDIR/${SAMPLE}.filt.bam
picard MarkDuplicates I=$BAMDIR/${SAMPLE}.filt.bam \
    O=$BAMDIR/${SAMPLE}.dedup.bam M=$BAMDIR/${SAMPLE}.metrics \
    REMOVE_DUPLICATES=true
samtools index $BAMDIR/${SAMPLE}.dedup.bam

# Tn5 shift: +4 / -5. alignmentSieve from deepTools does this in one step
alignmentSieve -b $BAMDIR/${SAMPLE}.dedup.bam \
    -o $BAMDIR/${SAMPLE}.shift.bam --ATACshift -p 8
samtools sort -@ 8 $BAMDIR/${SAMPLE}.shift.bam -o $BAMDIR/${SAMPLE}.shift.sort.bam
samtools index $BAMDIR/${SAMPLE}.shift.sort.bam

bamCoverage -b $BAMDIR/${SAMPLE}.shift.sort.bam \
    -o $BWDIR/${SAMPLE}_reads_count.bw --binSize 10 -p 8
