OTX2 case study - Fig 3b¶

Top 5 TF motifs enriched in 4C OTX2 CUT&RUN peaks (HOMER known motif enrichment, one-sided binomial).

Paper reports OTX2, GSC, DUX4, CRX, PHOX2A as the top 5 (P-values 10^-4544 to 10^-598). Exact ranks and P-values depend on the HOMER motif-database version. Our run recovers the same TF family (Homeobox, TAATCC-core recognition) in the top hits.

Paper: Wang, Q. et al. Maternal factor OTX2 regulates human embryonic genome activation and early development. Nature Genetics 57, 2772-2784 (2025). 10.1038/s41588-025-02350-8

%%script false --no-raise-error
# ==========================================================
# CUT&RUN FASTQ -> deduplicated BAM -> RPKM-normalised bigwig
# ==========================================================
# Reference: Wang et al. 2025 Methods ("CUT&RUN assay" section) +
# filename convention in OMIX006476 (`..._pmu_100bp_rpkm_bamCoverage.bw`).
#
# Raw FASTQs are hosted behind controlled access at GSA-Human HRA006621
# (paper Data Availability). For each replicate:
SAMPLE=h_OTX2_day2_CUTRUN_rep1         # one of rep1 / rep2 / KD etc.
FQDIR=/scratch/users/steorra/data/otx2/HRA006621/fastq
BAMDIR=/scratch/users/steorra/data/otx2/OMIX006476/bam
BWDIR=/scratch/users/steorra/data/otx2/OMIX006476
IDX=/scratch/users/steorra/data/hg19/bt2_idx/hg19
mkdir -p $BAMDIR $BWDIR

# 1. Adapter / quality trim
trim_galore --paired --gzip -o $BAMDIR/trim \
    $FQDIR/${SAMPLE}_R1.fastq.gz $FQDIR/${SAMPLE}_R2.fastq.gz

# 2. Bowtie2 alignment (paired, CUT&RUN-optimised)
bowtie2 --end-to-end --very-sensitive --no-mixed --no-discordant \
        -I 10 -X 700 -p 8 -x $IDX \
        -1 $BAMDIR/trim/${SAMPLE}_R1_val_1.fq.gz \
        -2 $BAMDIR/trim/${SAMPLE}_R2_val_2.fq.gz |
    samtools view -bS -q 30 -f 2 - |
    samtools sort -@ 8 -o $BAMDIR/${SAMPLE}.raw.bam

# 3. Picard MarkDuplicates -> "pmu" (picard mark unique) BAM
picard MarkDuplicates \
    I=$BAMDIR/${SAMPLE}.raw.bam \
    O=$BAMDIR/${SAMPLE}.pmu.bam \
    M=$BAMDIR/${SAMPLE}.pmu.metrics REMOVE_DUPLICATES=true
samtools index $BAMDIR/${SAMPLE}.pmu.bam

# 4. Merge biological replicates (rep1 + rep2 -> rep12)
samtools merge -@ 8 -o $BAMDIR/h_OTX2_day2_CUTRUN_rep12.pmu.bam \
    $BAMDIR/h_OTX2_day2_CUTRUN_rep1.pmu.bam \
    $BAMDIR/h_OTX2_day2_CUTRUN_rep2.pmu.bam
samtools index $BAMDIR/h_OTX2_day2_CUTRUN_rep12.pmu.bam

# 5. RPKM bigwig (matches published file name)
bamCoverage -b $BAMDIR/h_OTX2_day2_CUTRUN_rep12.pmu.bam \
    -o $BWDIR/h_OTX2_day2_CUTRUN_rep12_pmu_100bp_rpkm_bamCoverage.bw \
    --binSize 100 --normalizeUsing RPKM -p 8

Motif enrichment via epi.bulk.find_motifs_genome (pure-Python, GPU)¶

Peak BED → GC-matched background (50k random genomic windows) → batched PWM scan on GPU (conv1d over one-hot sequences) → right-tailed binomial with arbitrary-precision tail for extreme p-values. Uses HOMER's own motif library as the PWM source so top hits are directly comparable to findMotifsGenome.pl output. Runs in ~2 min end-to-end on an H100 (vs ~15 min for the Perl pipeline).

In [6]:
peaks = pd.read_csv(OMIX / 'h_OTX2_day2_CUTRUN_rep12_peaks.bed',
                    sep='\t', header=None,
                    names=['chrom','start','end','name','score'])

enrich = epi.bulk.find_motifs_genome(
    peaks, genome=HG19_FA, motif_library=MOTIF_DB,
    size=200, n_bg=50_000, seed=0,
    backend='auto',          # GPU if torch.cuda available, else CPU mp
    outdir=HOMER_OUT,        # writes HOMER-shaped knownResults + PWMs
    verbose=False,
)
